文章将会介绍到Git分布式版本控制工具，基于开发案例详细介绍Git常见命令及Git分支，其中包括分支特性、分支创建、分支转换、分支合并、代码合并冲突解决。在最后将会介绍常用开发工具IDEA集成Git

Git概述

Git是一个免费的、开源的分布式版本控制系统，可以快速高效的处理从小型到大型的各种项目

Git易于学习，占地面积小，性能极快。它具有廉价的本地库，方便的暂存区域和多个工作流分支等特性。其性能优于Subversion、CVS、Perforce和ClearCase等版本控制工具

分布式版本控制工具和集中化版本控制工具

集中化的版本控制系统：诸如CVS、SVN等，都有一个单一的集中管理的服务器，保存所有文件的修订版本，而协同工作的人们都通过客户端连接到这台服务器，取出最新的文件或者提交更新

分布式的版本控制系统：诸如Git、Mercurial、Bazarr、Darcs…类似Git这类分布式版本控制工具，客户端提取的不是最新版本的文件快照，而是把代码仓库完整的镜像下来（本地库）。这样任何一处协同工作用的文件发生故障，事后都可以用其他客户端的本地仓库进行恢复

Git的工作机制

Git一共分三个部分：工作区（写代码）—-git add—>暂存区（临时存储）—-git commit—>本地库（历史版本）

Git和代码托管中心

代码托管中心是基于网络服务器的远程代码仓库，一般称为远程库

  • 局域网：GitLab
  • 互联网：GitHub、Gitee

Git命令

命令名称 作用
git config --global user.name 用户名 设置用户签名
git config --global user.email 邮箱 设置用户签名
git init 初始化本地库
git status 查看本地库状态
git add 文件名 添加到暂存区
git commit -m "日志信息" 文件名 提交到本地库
git reflog 查看历史记录
git reset --hard 版本号 版本穿梭

Git分支

分支的概述和优点

分支概述

在版本控制过程中，同时推进多个任务为每个任务，我们就可以创建每个任务的单独使用分支意味着程序员可以把自己的工作从开发主线上分离开来，开发自己分支的时候，不会影响主线分支的运行。对于初学者而言，分支可以简单理解为副本，一个分支就是个单独的副本。（分支底层其实也是指针的引用）

分支优点

  • 同时推进多个功能开发，提高开发效率

  • 各分支在开发过程中，如果某一个分支开发失败，不会对其他分支有任何影响。失败的分支删除重新开始即可

分支的操作

命令名称 作用
git branch 分支名 创建分支
git branch -v 查看分支
git checkout 分支名 切换分支
git merge 分支名 把指定的分支合并到当前分支上

合并冲突问题：合并分支时，在两个分支同一个文件的同一个位置有两套完全不同的修改，Git无法做出判断使用哪一个时，必须人为决定新代码内容

手动修改完代码后，需要重新add、commit，在提交时不需要加文件名，且提交后在当前分支内容会改变

Git团队协作机制

部门内的协作：push、clone、pull

部门间协作：push、clone、pull、fork、pull request、merge

GitHub操作

https://github.com/

命令名称 作用
git remove -v 查看当前所有远程地址别名
git remote add 别名 远程地址 起别名
git push 别名 分支 推送本地分支上的内容到远程仓库
git clone 远程地址 将远程仓库的内容克隆到本地
git pull 远程地址别名 远程分支名 将远程仓库对于分支最新内容拉下来后与本地库合并

IDEA集成Git

配置Git忽略文件：

与项目实际功能无关，不参与服务器上部署运行。把他们忽略掉能够屏蔽IDE工具之间的差异

  • 创建xxx.ignore（建议使用git.ignore）

  • 将文件放在用户家目录下

  • 在.gitconfig文件中引用忽略配置文件

    这里使用正斜杠（’/‘）

IDEA定位Git程序：

IDEA工具——->settings——->Version Control——->Git——->Path to Git executable中选择Git的安装目录（git.exe）

IDEA中初始化本地库：

工具栏VCS——->Import into Version Control——->Create Git Repostiory

IDEA中使用Git Add：

文件名右键——->Git——->Add

当文件过多时不用依次手动Add，可以直接右键点击真个项目总目录添加，再添加时忽略文件

IDEA代码切换版本：

在IDEA左下角的Version Control——->Log中查看提交信息

当需要切换版本时，右键点击版本选择Checkout Revision ‘版本号’

IDEA创建分支：

方式一：右键点击项目名称——->Git——->Repository——->Branches

方式二：点击IDEA右下角的Git:分支名，创建新的分支

IDEA合并分支：

正常合并：点击IDEA右下角的Git:分支名——->点击需要合并的分支——->Merge into Current

冲突合并：点击IDEA右下角的Git:分支名——->点击需要合并的分支——->Merge into Current——->点击Merge手动进行合并

IDEA集成GitHub

检查是否存在GitHub插件：IDEA工具——->settings——->Version Control——->GitHub

如果IDEA不存在GitHub插件，在settings——->Plugins上下载插件

设置GitHub账号：

账号登录：IDEA工具——->settings——->Version Control——->GitHub——->’+’

Use Token（口令登录）：在GitHub账号——->settings——->Developer settings——->Personal access token——->Generate new token

分享项目到GitHub仓库：（创建远程库并Push代码到远程库）

工具栏VCS——->Import into Version Control——->Share Project on GitHub

IDEA中使用Push：

通过https：首先提交到本地库——->工具栏VCS——->Git——->Push

通过ssh：Define Remote自定义ssh连接

IDEA中使用Pull：

工具栏VCS——->Git——->Pull

IDEA中使用Clone：

在IDEA初始化页面——->Get from Version Control
